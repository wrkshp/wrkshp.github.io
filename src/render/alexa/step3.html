Skill Configuration

Once you've clicked into the Configuration tab, we'll need to configure the endpoints. This means we have to build our ARN, or Amazon Resource Name. We'll have to use AWS for this.

  1. Navigate to aws.amazon.com.
  2. Change your location at the top right to N.Virginia. This is the only physical servers that supports Alexa Skills thus far.
  3. Make sure you've verified your email with your AWS account. If you haven't, there may be errors that show up that don't seem to make sense.
  4. Navigate to Lambda.

Build your Lambda Function

Lambda is a really useful tool that Amazon Web Services (AWS) has built to allow you to run a service without provisioning your own servers. Here, we'll be using Lambda to store and run our Skills code.

We'll be creating a Lambda function using Python 3.6. There may be some blueprint, or template, for an Alexa skill for other languages, and you can explore these if you'd like. However, we'll be starting with a blank one, since Python 3.6 doesn't have an Alexa blueprint.

Now, we have a trigger, which is the event that launches the Lambda function. In our case, we'll select the Alexa Skills Kit. Click continue, and we'll have some Lambda information we'll have to configure:

  • Name is the lambda function name. You can name this whatever you like.
  • Description can be whatever you want as well.
  • Runtime. We'll be using Python 3.6.
