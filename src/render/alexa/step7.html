Configuring Lambda

Upload Your Code

Now, we'll navigate to Lambda in our AWS console again.

Under the Code tab, we'll paste the code that we have now. You can ignore the Environment Variables for now.

Configure your Lambda

Click the Configuration tab. Here we'll fill out a few things.

  • Runtime must be whatever language your code is written in. In our case, it's Python 3.6.
  • Handler is the entry point, or the main function that we have to call. In our case, we'll fill this out with lambda_function.lambda_handler, which is made up of two things:
    • lambda_function refers to the fact that we used the "inline" option to submit our code, and thus only have one file. If you build a more complicated skill, you'll have more files to choose from, so in that case, you would replace this with the filename.
    • lambda_handler refers to the function name in the lambda_function that we specified. In our case, we've called our function lambda_handler, which is by default.
  • Role is used to specify the permissions function. While we won't be diving into what this is exactly, you can learn more here. For our purposes, we'll be creating a new role from a template.
  • Role Name is the name of the roll. You can name this whatever you want, but we'll be calling ours AlexaSkill.
  • Policy Template is simply a template for our permissions that we have to define. We won't be diving into this, but you can learn more here. We only need Simple Microservice permissions, so select that.
  • Description describes your skill. Use whatever you want.

Click save. You've successfully created your Lambda function!

ARN

Now remember the reason we came here was because we were looking for an ARN, or Amazon Resource Name. This should be at the top right corner of your page. A couple things to make sure of: your location is in N.Virginia, and your ARN includes us-east-1 in it. Remember, this is the only physical location in AWS for Alexa Skills so far. Your skill won't work without this configured properly.

Navigate back to developer.amazon.com, and in the Configuration tab, paste in the ARN. We don't have to worry about the rest of this page. Click next!
